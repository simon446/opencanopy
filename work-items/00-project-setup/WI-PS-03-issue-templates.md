## Objective

Make it easy for contributors across all five disciplines to file structured, triageable issues.

## Deliverables

- [ ] Issue templates under `.github/ISSUE_TEMPLATE/`: bug, firmware, hardware/PCB, mechanical, docs.
- [ ] Each template prompts for: affected track, spec section, hardware revision, repro/steps.
- [ ] `CONTRIBUTING.md` covering branch naming, PR review per track, and the test-before-merge rule.
- [ ] PR template referencing the relevant work-item ID and its acceptance checklist.

## Acceptance criteria

- A bug, hardware, firmware, mechanical, and docs template each exist and render on GitHub/GitLab.
- PRs are required to cite a work-item ID.
